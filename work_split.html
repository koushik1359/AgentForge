AgentForge — Work Split Koushik & Siri | Dynamic AI Agent Builder Platform

Koushik — The Orchestration Layer

Koushik

Owns the brain of the system — the parts that receive user input and coordinate all other agents.

Agents

AgentWhat It Does
Agent Builder AgentMain orchestrator — receives user request, calls all other agents in sequence
Goal Analysis AgentParses user intent, extracts objective, frequency, and output type
Validation AgentChecks the final generated agent for correctness before deployment

Other Responsibilities

FastAPI backend Azure Container Apps deployment Agent creation form (Frontend)

Learning Focus

How to build an orchestrator — a LangGraph agent that calls other agents as tools. The most advanced and most important pattern in the system.

Siri — The Specialist Agents Layer

Siri

Owns all the agents that do the actual specialized work when called by the orchestrator.

Agents

AgentWhat It Does
Tool Selection AgentPicks the right tools (Gmail API, Search, AWS, etc.) for the generated agent
Workflow Design AgentBuilds the step-by-step execution graph using LangGraph
Prompt Generation AgentWrites the system prompt and role instructions for the generated agent
Memory Configuration AgentDecides what the agent should remember across runs

Other Responsibilities

PostgreSQL schema (Neon) pgvector memory setup Supabase Auth integration Agent dashboard (Frontend)

Learning Focus

How to build specialist agents — focused, single-purpose LangGraph nodes that take structured input and return structured output.

Daily Git Workflow (Both Follow This)

# Start of every session
git pull

# While working — commit often
git add .
git commit -m "feat: describe what you just built"

# End of session
git push
Since each person owns different files, you will almost never have a merge conflict. Both push directly to main.
